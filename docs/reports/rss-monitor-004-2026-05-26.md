## Candidate Feeds Considered

Selected:

- `BOC_CANARIAS`: official BOC RSS subscriptions page lists section feeds; selected the official
  `I. Disposiciones generales` RSS feed because it returned HTTP 200 and parsed as RSS.
- `DOG`: official Diario Oficial de Galicia RSS subscriptions page lists `Sumario completo`; selected
  the complete summary RSS feed because it returned HTTP 200 and parsed as RSS.
- `BOP_LUGO`: official bulletin landing page exposes `RSS BOP`; selected the official feed because it
  returned HTTP 200 and parsed as RSS.

Rejected or deferred:

- `BORM`: the official web surface declares RSS and an Angular RSS service, but direct fetches to the
  discovered RSS/service paths returned a Radware captcha page in this environment.
- `BOP_SORIA`: a discovered RSS path parsed, but the sampled item had weak metadata
  (`title=NULL`, `published_at=NULL`), so it was not promoted in this task.
- `BOP_AVILA`: a discovered RSS path parsed, but sampled entries link directly to PDF documents; this
  is better reviewed in a separate PDF-first metadata task.
- `BOP_BADAJOZ`: official Atom feed parsed, but sampled entries lacked `published_at`; deferred
  because three better bounded sources were available.
- `BOP_VALLADOLID`: official RSS path parsed, but sampled entries lacked `published_at`; deferred
  because three better bounded sources were available.

## Safety Boundary

This task used preview mode only. It did not use `--write`.

It did not:

- create `data/rss_monitor`;
- create candidates;
- create evidence-grade records;
- download PDFs;
- write artifacts;
- write downstream data;
- run backfills;
- run broad/all-source monitoring;
- touch VPS/prod DB;
- touch Hermes;
- touch systemd.

`BOP_LUGO` feed summaries can contain PDF URLs, but they remain metadata in the RSS record. The
monitor does not fetch those URLs.
