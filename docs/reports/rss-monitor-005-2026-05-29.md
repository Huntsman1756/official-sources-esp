## Parser Note

`BOP_BADAJOZ` Atom entries do not expose Atom `published` in the sampled feed. The monitor now uses
Atom `updated` as the primary entry date when `published` is absent, preserving the metadata-only
contract and avoiding source-specific parsing.

## Safety Boundary

This task used preview mode only. It did not use `--write`.

It did not:

- create `data/rss_monitor`;
- create candidates;
- create evidence-grade records;
- download PDFs;
- write artifacts;
- write downstream data;
- run backfills;
- run broad/all-source monitoring;
- touch VPS/prod DB;
- touch Hermes;
- touch systemd.

BOCM RSS summaries can contain PDF URLs as text metadata. The monitor does not fetch those URLs.
